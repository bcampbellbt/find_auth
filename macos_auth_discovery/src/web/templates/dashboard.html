Understanding Status Values

The Status column shows the current configuration state of each authorization point:

  • enabled/disabled: Whether the security feature is currently active
  • configured: The feature has been set up and is available
  • available: The feature is present but may not be actively configured
  • protected/secured: The item is secured and requires authorization to access
  • requires_admin: Administrative privileges are needed to modify
  • restricted: Access is limited by system policies
